# Инструкция: как обновить сайт после изменений

Когда вы вносите изменения в код или контент сайта, нужно загрузить их на GitHub — Vercel автоматически обновит сайт.

---

## Шаг 1: Сохранить изменения локально

После правок в Cursor сохраните все изменённые файлы (Ctrl+S).

---

## Шаг 2: Добавить изменения в Git

Откройте терминал (PowerShell) в папке проекта и выполните:

```powershell
cd d:\UHome
git add .
git status
```

`git status` покажет, какие файлы будут загружены.

---

## Шаг 3: Сделать коммит

```powershell
git commit -m "Описание изменений"
```

**Примеры описания:**
- `"Обновлён текст на главной"`
- `"Добавлены новые фото в портфолио"`
- `"Изменены цены в калькуляторе"`
- `"Исправлены контакты"`

---

## Шаг 4: Отправить на GitHub

```powershell
git push
```

---

## Шаг 5: Дождаться деплоя

1. Зайдите на [vercel.com/dashboard](https://vercel.com/dashboard)
2. Откройте проект **uhome**
3. В разделе **Deployments** увидите новый деплой (со статусом Building → Ready)
4. Обычно это занимает 1–2 минуты

Когда статус станет **Ready**, сайт обновлён и изменения видны всем.

---

## Краткая памятка (скопировать и выполнять по порядку)

```powershell
cd d:\UHome
git add .
git commit -m "Описание изменений"
git push
```

---

## Если что-то пошло не так

| Проблема | Решение |
|----------|---------|
| `nothing to commit` | Изменения не сохранены или не попали в `git add` |
| `failed to push` | Проверьте интернет, повторите `git push` |
| Сайт не обновился после push | Подождите 2–3 минуты, обновите страницу с Ctrl+F5 |
| Ошибка при деплое в Vercel | Откройте логи деплоя в Vercel → Deployments → клик по деплою |

---

## Где лежат основные файлы для правок

| Что менять | Папка/файл |
|------------|------------|
| Тексты главной страницы | `site/app/page.tsx`, `site/components/sections/` |
| Услуги | `site/data/` или `site/lib/mock-data.ts` |
| Портфолио | `site/lib/mock-data.ts` |
| Калькулятор, цены | `site/data/price_list.json`, `site/lib/calculator.ts` |
| Контакты, конфиг | `site/lib/site-config.ts` |
| Стили, цвета | `site/app/globals.css`, `site/tailwind.config.ts` |
| **Настройка форм, email** | `site/ИНСТРУКЦИЯ_FORMSPREE.md` |
